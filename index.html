My essays

My last post's depend on each other, and it should be read in sequence. Articles not in bold are optional.

  • Assignment Statement Considered Harmfull. Pervasive mutable state makes programs more difficult to deal with. While this is old news, most programmers don't know or don't care. It's a pity because simple remedies exists.
  • Defining syntatic sugar. I sometime hear arguments about what is or is or is not syntactic sugar. This is an attempt to define it.
  • Class-bases Programming as Syntactic Sugar. Most of the time, "Object Oriented" actually mean classes. Classes are nothing more then syntactic sugar, at least in statically typed language such as Java and C++. Sugar is not useless, but seeing past it helps us understand classes better.
  • Taboo "OO". The term "OO" is so overloaded that we should stop using it. Better substitues are "classes" and "prototypes".
  • How Class based Programming Sucks. Classes are sub-optimal. Functional programming is far better.
